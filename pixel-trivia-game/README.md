# Pixel Trivia Game - 像素復古問答車輪戰

這是一個採用 `React Vite` 和 `Google Apps Script` 打造的「2000 年代街機像素風」網頁問答遊戲。
透過完全免費的 Google Sheets 作為輕量級資料庫及後台管理系統，你能隨時新增或修改題目，並自動紀錄每位闖關玩家的成績。

---

## 🚀 系統架構簡介

* **前端 (Frontend)**: React + Vite + Vanilla CSS
* **後端 (Backend)**: Google Apps Script (處理 API 請求與計分運算)
* **資料庫 (Database)**: Google Sheets (題目管理與玩家分數紀錄)
* **關主圖片產生器**: DiceBear API (基於 Seed 生成獨特像素頭像)

---

## 🛠️ 第一部分：設定 Google Sheets 與 Backend API

這一步最為重要，需要你親自在 Google 帳號中設定資料庫與部署 API。

### 1. 建立你的 Google 試算表 (Database)
1. 登入 Google 雲端硬碟，新增一份 **空白的 Google 試算表**。
2. 將第一頁工作表命名為 **`題目`**。
3. 在 `題目` 表格的 **A1 到 G1** 輸入以下表頭（請務必一字不亂）：
   `題號`、`題目`、`A`、`B`、`C`、`D`、`解答`
4. 將第二頁工作表命名為 **`回答`**。
   *(這頁不需要手動建立表頭，程式未來會自動幫你建置對應的成績紀錄欄位)*

### 2. 部署你的 Google Apps Script (API)
這步會將你的試算表變成一支可以讓遊戲讀取資料與讀寫成績的「網址 API」。

1. 回到剛建立好的試算表，點擊最上方的 **擴充功能** ➡️ **Apps Script**。
2. 系統會跳出一個程式碼編輯器。請將編輯器原本預設的 `function myFunction()` 程式碼**全部刪除**。
3. 將本專案附帶的 `gas_script.js` 裡面的所有內容 複製並**貼上**至編輯器中。
4. 點擊上方的 **💾 儲存專案**。
5. **開始部署**：點擊右上角藍色的 **部署** ➡️ 選擇 **新增部署作業**。
6. 設定部署作業形式：
   * 點擊左側的「選取類型 (齒輪圖示)」並選擇 **網頁應用程式 (Web app)**。
   * 新增說明填寫 `v1.0 部署`。
   * 「誰可以存取」：**務必選擇「所有人 (Anyone)」**。
   * 點擊下方的 **部署** (可能需要授權 Google 權限，請點進階 -> 信任前往)。
7. 部署完成後，畫面會跳出一串「網頁應用程式的網址」，**請將這串 URL 複製下來** (這是你後續要放進專案裡的秘密武器)。

> **⚠️ 更新注意 - 非常重要**
> 未來你只要有對 Apps Script 程式碼做任何修改，點選儲存後，務必點擊右上角的 **部署 ➡️ 管理部署作業 ➡️ 編輯 (鉛筆) ➡️ 版本選擇「建立新版本」 ➡️ 部署**。否則你的修改將永遠不會生效在 API 網址上。

## 💻 第二部分：安裝與設定 React Frontend

### 1. 安裝環境依賴包
請確保你的電腦已經安裝 [Node.js](https://nodejs.org/)。開啟終端機，進入專案資料夾後輸入：
```bash
npm install
```

### 2. 設定環境變數 (.env)
在專案根目錄，找到或者新建名為 `.env` 的檔案，將你剛才在 Google Apps Script 複製的網址放進去。你可以隨時調整要抽幾題與過關門檻。

```env
# 你的 Apps Script 部署 URL
VITE_GOOGLE_APP_SCRIPT_URL=https://script.google.com/macros/s/.../exec

# 玩家需要答對幾題才算過關
VITE_PASS_THRESHOLD=3

# 每次遊戲從題庫抽幾題出來玩
VITE_QUESTION_COUNT=5
```

### 3. 啟動本機開發伺服器
```bash
npm run dev
```
打開瀏覽器，前往 `http://localhost:5173/`，即可馬上測試你的像素風大挑戰！

## 🌐 第三部分：透過 GitHub Pages 免費自動部署上線

如果你希望把這個遊戲放到網路上給所有人玩，可以利用本專案內建的 **GitHub Actions** 流程，幾分鐘內就能自動完成打包與部署！

### 1. 將專案推送到 GitHub
將目前的檔案與程式碼建立一個 Git 儲存庫，並推送到你的 GitHub 帳號下（建議設為 `Public` 才能免費使用 GitHub Pages）。

### 2. 開放 GitHub Pages 的讀寫權限
1. 進入你的 GitHub Repo 頁面，點選上方的 **Settings**。
2. 左側選單選擇 **Actions** > **General**。
3. 滑到最下面找到 **Workflow permissions**，選擇 **Read and write permissions**，並按下 **Save** 保存。
4. 左側選單選擇 **Pages**，在 **Build and deployment** 的 `Source` 欄位選取 **GitHub Actions**。

### 3. 設定專案的環境變數秘密 (Repository Secrets)
因為我們不在 GitHub 上公開這支對外連線的 API 網址，你需要把它們存入 Secrets 中讓 Actions 讀取 (變數名稱請參考 `.env.example`)。
1. 進入你的 Repo 點選 **Settings**。
2. 左側選單選擇 **Secrets and variables** > **Actions**。
3. 點擊綠色按鈕 **New repository secret**，依序新增以下三個：
   * `VITE_GOOGLE_APP_SCRIPT_URL`：貼上你剛產生的 Apps Script 部署網址（**必填**）。
   * `VITE_PASS_THRESHOLD`：填入過關的最低分數 (例如填數字 `3`)。
   * `VITE_QUESTION_COUNT`：填入每次題庫隨機抽幾題 (例如填數字 `5`)。

### 4. 觸發自動部署
一切設定就緒後，未來只要你有任何修改推送到 `main` 分支，GitHub 就會自動幫你打包並發布新版網頁！
如果你想立刻手動發布第一版：
1. 點擊上方的 **Actions** 頁籤。
2. 點選左側名為 **Deploy to GitHub Pages** 的流程。
3. 點擊右側的 **Run workflow** -> 綠色按鈕 **Run workflow** 即可觸發執行！
4. 等待打勾後，你的專案專屬網址就能直接對外公開分享囉！
